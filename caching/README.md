# Caching

Caching concepts, patterns, and service-specific deep dives.

## Documentation Safety and Source Policy

- No hardcoded secrets, keys, tokens, passwords, or connection strings are included.
- All command snippets use placeholders like `<rg>`, `<ip>`, `<password>`.
- Content is based on public Redis documentation and Microsoft Learn.
- Do not paste real credentials into markdown files.

## Learning Flow (Recommended)

1. Caching fundamentals and workflows
2. Redis data structures and internals
3. Persistence (RDB, AOF, hybrid)
4. Replication and high availability (Sentinel)
5. Cluster and horizontal scaling
6. Transactions, Lua, and pipelining
7. Design patterns (cache-aside, write-through, distributed locks, rate limiting)
8. Azure Cache for Redis tiers and operations
9. Performance tuning and observability
10. Security (ACL, TLS, network hardening)
